# **Spring Core**

## 1. IoC (Inversion of Control)
-   `IOC means spring creates beans for us and gives us, instead of we creating mannually using new keyword.`
-   Concept of giving control of object creation to Spring.

    ### Why IOC?
    - Reduces mannual object creation.
    - Makes code loosely coupled.
    - Spring manages object lifecycle.

    ### How IOC works?
    - Spring scans annotated classes.
    - Creates beans.
    - Stores them inside the container.
    - Injects them whenever required. (Using `@Autowired`)


## 2. IoC Container

-   `Thing that implements the IOC idea.` **(IoC Container creates beans and manages them)**
-   Types:
    -   BeanFactory
    -   ApplicationContext


## 3. Dependency Injection (DI)

* Ways Spring injects dependencies: 
    - Setter Injection 
    - Constructor Injection 
    - Field Injection


## 4. Bean
#### 👉 `Object created and managed by the Spring IoC container are called beans.`
-   Bean lifecycle management.

## 5. Bean Scope
#### 👉 `Bean scope means how many beans should spring create and how long they live.`
* **Common scopes:**
    - `singleton` (default) → same object every time
    - `prototype` → new object every time
    - `request` → one object per HTTP request
    - `session` → one object per session
    
* **Example:** `<bean id="student" class="Student" scope="prototype"/>`

## 6. Bean Configuration
#### 👉 `Procedure of defining and creating beans so that Spring IoC container can create and manage them.`
* Ways to define beans: 
    1. **XML config**
    2. **Annotation config** 
    3. **Java config (`@Configuration`, `@Bean`)**

    ### **1. XML config**
    * **We can perform `Dependency Injection` using `XML configuration` in 2 ways.**
        1. **Setter Injection -**
            * Value Injection:
                ``` 
                <bean id="student" class="Student">
                    <property name="name" value="Rahul"></property>
                </bean>

                This injects primitive/String value.
                ```

            * Object Injection:
                ```
                <bean id="student" class="Student">
                    <property name="address" ref="gcAddress"></property>
                </bean>

                This injects another bean object.
                ```  
                
        2. **Constructor Injection -**
            * Value Injection:
                ```
                <bean id="student" class="Student">
                    <constructor-arg name="name" value="Rahul"></constructor-arg>
                </bean>
                ```

            * Object Injection:
                ```
                <bean id="student" class="Student">
                    <constructor-arg name="address" ref="gcAddress"></constructor-arg>
                </bean>
                ```


## 7. Autowiring

* Automatic dependency injection.
* Common annotations: - `@Autowired` - `@Qualifier` - `@Primary`

## 8. Component Scanning

* Spring automatically scans packages to detect components.

* Annotations: - `@Component` - `@Service` - `@Repository` - `@Controller`

## 9. Bean Lifecycle

* Bean initialization and destruction.

* Common methods/annotations: 
    - `init()` 
    - `destroy()` 
    - `@PostConstruct` 
    - `@PreDestroy`
