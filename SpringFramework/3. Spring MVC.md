# Spring MVC

### 1. Why does Spring MVC use DispatcherServlet?
- `DispatcherServlet implements the Front Controller pattern and acts as a central entry point for all HTTP requests, delegating them to appropriate controllers.`

    | Component                      | Responsibility                                                                                                                                                                                                                                            |
    | ------------------------------ | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
    | **Servlet Container (Tomcat)** | Hosts the web application and manages servlets. It receives the HTTP request from the browser and forwards it to the DispatcherServlet registered in the application.  
    | **DispatcherServlet**          | Central servlet of Spring MVC. Receives all HTTP requests from the servlet container (Tomcat) and coordinates the entire request processing workflow. It delegates work to other Spring MVC components like HandlerMapping, Controller, and ViewResolver. |
    | **HandlerMapping**             | Determines which controller method should handle the incoming request. It checks the URL and HTTP method against mappings such as `@GetMapping`, `@PostMapping`, `@RequestMapping`.                                                                       |
    | **Controller**                 | Contains the business logic that processes the request. It interacts with services or repositories if needed, prepares data, and returns a view name or response.                                                                                         |
    | **Model**                      | A container used to transfer data from the controller to the view. Data is stored using `addAttribute()` and later accessed in the view template. Internally it behaves like a `Map<String, Object>`.                                                     |
    | **ViewResolver**               | Resolves the logical view name returned by the controller into an actual view file. For example `"hello"` becomes `templates/hello.html` when using Thymeleaf.                                                                                            |
    | **View (Thymeleaf/JSP)**       | The template that generates the final HTML response. It reads data from the model and renders dynamic content using expressions like `${name}`.                                                                                                           ||

### 2. Difference between @Controller and @RestController
* **What is @RestController?**
    - `@RestController is used to create REST APIs.`
    - **It tells Spring:**
        ```
        Return data directly in the HTTP response body
        ```
        - instead of returning a view (HTML page).

    * **@RestController is actually a combination of:** <br>
        `@Controller + @ResponseBody`

    #### Example comparison
    -   Ex1:
        ```
        @Controller
        public class DemoController {

            @GetMapping("/hello")
            public String hello() {
                return "hello";
            }
        }
        ```

    * **Spring searches for:** `templates/hello.html`


    *   Ex2:
        ```
        @RestController
        public class DemoController {

            @GetMapping("/hello")
            public String hello() {
                return "Hello World";
            }
        }
        ```

    * **Spring returns:** `Hello World`
        directly to the client.
    <br><br>

    | Feature              | `@Controller`   | `@RestController`      |
    | -------------------- | --------------- | ---------------------- |
    | Purpose              | Web pages (MVC) | REST APIs              |
    | Return value         | View name       | Data (JSON/XML/String) |
    | ViewResolver used    | Yes             | No                     |
    | @ResponseBody needed | Yes             | No                     |
