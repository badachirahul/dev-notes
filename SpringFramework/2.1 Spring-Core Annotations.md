# Spring-Core Annotations:
## 1. Stereotype annotations (create beans)
* `Stereotype annotations are used to mark classes as Spring beans so that the IoC container can automatically detect and register them.`
* **Tpes:** <br>
    `1. @Component`<br>
    `2. @Service`<br>
    `3. @Repository`<br>
    `4. @Controller` **(belongs to Spring MVC)**<br>

    1.  **@Component -**
        * Basic stereotype annotation.
        * Used to tell Spring: `Create a bean of this class.`

    2. **@Service -**
        * Used for service layer (business logic).
        * Purpose:
            - Business logic
            - validation
            - processing
        * **Technically it behaves like @Component, but used for better architecture.**

    3. **@Repository -**
        * Used for DAO / database layer.
        * `Extra feature: Spring automatically converts database exceptions into Spring exceptions.`

    4. **@Controller -**
        * Used for handling web requests.
* **`We can use @Component instead of all 3 stereotype annotations, but it is not recommended for architecture clarity and framework features.`**


## 2. Dependency Injection annotations
* **We can inject the dependencies in 3 ways:** <br>
    `1. @Autowired`<br>
    `2. @Qualifier`<br>
    `3. @Primary`<br>

    1. **@Autowired -**
        * **It tells Spring:** `Find a matching bean and inject it here.`
        * **Placed at the:** `Injection point.`


    2. **@Qualifier -** 
        * Used when multiple beans of same type exist.
        * **Placed at the:** `Injection point.`
        * **Ex:**
            ```
            @Component
            class Address {}

            @Component
            class OfficeAddress extends Address {}
            ```
            * Now Spring gets confused.

            ```
            @Autowired
            @Qualifier("officeAddress")
            Address address;
            ```
            * `@Qualifier() tells Spring which bean to inject when multiple beans of the same type exist, but injection is performed by @Autowired`

    3. **@Primary -**
        * Used to define default bean when multiple beans exist.
        * **Placed on the:** `Bean class`
        * Situation: Multiple beans of same type
            ```
            Now Spring has two beans of type Address.

            @Component
            class Address {}

            @Component
            @Primary
            class OfficeAddress extends Address {}
            ```
            ```
            Injection:

            @Autowired
            Address address;
            ```

            ```
            Spring checks:

            Multiple Address beans found
            ↓
            Which one should I inject?
            ↓
            Check for @Primary
            ↓
            OfficeAddress is marked @Primary
            ↓
            Inject OfficeAddress
            ```
