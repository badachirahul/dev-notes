# **Spring-Core Annotations:**
## 1. Stereotype annotations (create beans)
* `Stereotype annotations are used to mark classes as Spring beans so that the IoC container can automatically detect and register them.`
* **Tpes:** <br>
    `1. @Component`<br>
    `2. @Service`<br>
    `3. @Repository`<br>
    `4. @Controller` **(belongs to Spring MVC)**<br>

    1.  **@Component -**
        * Basic stereotype annotation.
        * Used to tell Spring: `Create a bean of this class.`

    2. **@Service -**
        * Used for service layer (business logic).
        * Purpose:
            - Business logic
            - validation
            - processing
        * **Technically it behaves like @Component, but used for better architecture.**

    3. **@Repository -**
        * Used for DAO / database layer.
        * `Extra feature: Spring automatically converts database exceptions into Spring exceptions.`

    4. **@Controller -**
        * @Controller tells Spring that this class is a web controller that handles HTTP requests.
        * **@Controller is used when you want to return:**
            - HTML pages
            - JSP
            - Thymeleaf templates
* **`We can use @Component instead of all 3 stereotype annotations, but it is not recommended for architecture clarity and framework features.`**

    <br>

    ### 👉 Spring Data JPA Repository (Short Notes)

    * Any interface that extends **JpaRepository** then spring will automatically detect.
    * Spring generates the implementation automatically.
    * A bean is created and registered in the IoC container.
    * So you can directly use:
        ```
        @Autowired
        StudentRepo repo;
        ```

    * `@Repository annotation is not required for Spring Data JPA repositories.`
    * The repository must be inside the component scan package.

## 2. Dependency Injection annotations
* **We can inject the dependencies in 3 ways:** <br>
    `1. @Autowired`<br>
    `2. @Qualifier`<br>
    `3. @Primary`<br>

    1. **@Autowired -**
        * **Find matching bean and inject it on the `field` or in `setter` or in `constructor`.**
        * **Placed at the:** `Injection point.`


    2. **@Qualifier -** 
        * **Used when multiple beans of same type exist and we have to choose a bean for injection.**
        * **Placed at the:** `Injection point.`
        * **Ex:**
            ```
            @Component
            class Address {}

            @Component
            class OfficeAddress extends Address {}
            ```
            * Now Spring gets confused.

            ```
            @Autowired
            @Qualifier("officeAddress")
            Address address;
            ```
            * `@Qualifier() tells Spring which bean to inject when multiple beans of the same type exist, but injection is performed by @Autowired`

    3. **@Primary -**
        * **Used to specify default bean when multiple beans of same type exists.**
        * **Placed on the:** `Bean class`
        * Situation: Multiple beans of same type
            ```
            Now Spring has two beans of type Address.

            @Component
            class Address {}

            @Component
            @Primary
            class OfficeAddress extends Address {}
            ```
            ```
            Injection:

            @Autowired
            Address address;
            ```

            ```
            Spring checks:

            Multiple Address beans found
            ↓
            Which one should I inject?
            ↓
            Check for @Primary
            ↓
            OfficeAddress is marked @Primary
            ↓
            Inject OfficeAddress
            ```

- `@Qualifier has higher priority than @Primary`

## 3. Configuration annotations
* **These annotations are used for Java-based configuration instead of XML.**
* **Types:** <br>
    `1. @Configuration` <br>
    `2. @Bean`<br>
        
    1. **@Configuration -**
        * **Marks a class as a Spring configuration class.**
        * **Example:**
            ```
            @Configuration
            public class AppConfig {

            }
            ```
            * Spring treats this class like XML configuration file.

    2. **@Bean -**
        * **Used inside a configuration class to create beans.**
        * **Example:**
            ```
            @Configuration
            public class AppConfig {

                @Bean
                public Student student() {
                    return new Student();
                }
            }
            ```
<br>

* **Difference between @Bean and @Autowired:**
    | `@Bean`                                                   | `@Autowired`                               |
    | --------------------------------------------------------- | ------------------------------------------ |
    | Used to **define a bean manually**                        | Used to **inject a bean**                  |
    | Defines a bean in the IoC container                       | Injects an existing bean into another bean |
    | Inside `@Configuration` class                             | Field / Constructor / Setter               |
    | **Bean creation**                                         | **Dependency injection**                   |
